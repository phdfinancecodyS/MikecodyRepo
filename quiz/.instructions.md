# Quiz Configuration Instructions

## 10 JSON Config Files

| File | Purpose |
|------|---------|
| `quiz-content.json` | 10 scored questions (0-3 each, 0-30 total range) |
| `topic-catalog.json` | 47 topics by domain |
| `topic-matcher-flow.json` | Post-quiz 3-question topic matcher (14 topicHints) |
| `audience-bucket-flow.json` | Audience identity/context matcher (17 buckets) |
| `base-guide-catalog.json` | 79 base guides with domains, tags, offer lanes |
| `recommendation-routing-config.json` | Risk-based routing pipeline |
| `product-catalog.json` | 4 products: Guide ($9), Kit ($19), SMS ($4.99/mo), Bundle ($34) |
| `fulfillment-config.json` | Post-purchase delivery actions per product |
| `conversation-branch-flow.json` | Post-quiz conversational branching |
| `api-contracts.json` | Machine-readable API request/response contracts |

## Scoring (Non-Negotiable)

- 0–10 = low_risk, 11–20 = moderate_risk, 21–25 = high_risk, 26–30 = critical
- Q5 = 3 → force critical
- Q5 = 2 → minimum high_risk
- Critical: crisis first, no paid offers above crisis CTAs, capture blocked before crisis actions

## Cross-References

- `base-guide-catalog.json` guide IDs must match `planning/GUIDE-OFFER-MAPPING.csv`
- `audience-bucket-flow.json` buckets must match `planning/AUDIENCE-SLANT-MANIFEST.csv`
- `topic-matcher-flow.json` topicHints must resolve against `topic-catalog.json`
- All 79 base_paths in `base-guide-catalog.json` must exist on disk under `content/topic-guides/`

## Validation

Run `python3 scripts/full_workspace_audit.py` — checks all JSON parsing, cross-references, and scoring logic.
