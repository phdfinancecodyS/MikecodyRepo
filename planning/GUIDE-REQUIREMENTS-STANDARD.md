# Guide Requirements Standard

Date: 2026-03-20
Status: Required for all new and revised guides

## Non-Negotiables

1. Plain human language, no academic tone.
2. One guide = one clear user problem.
3. Every guide includes 2 worksheets minimum.
4. Safety resources are always free and visible where relevant.

## Required Guide Structure

1. What this helps with (plain-language)
2. What is happening (short explanation)
3. What to do now (step-by-step)
4. What to say (script lines)
5. Common mistakes to avoid
6. 24-hour action plan
7. Two worksheets

## Worksheet Requirements (Every Guide)

Worksheet 1: Pattern Finder
- Purpose: identify triggers/signals/context
- Output: user names top 3 triggers and earliest warning sign

Worksheet 2: Action Builder
- Purpose: turn insight into behavior
- Output: one script + one specific 24-hour commitment

## Tone Requirements

Use:
- short sentences
- direct language
- validating but practical voice

Do not use:
- diagnosis-heavy labels as headline copy
- dense research language
- shame-based framing

## Productization Requirements

Each guide should map to one offer lane:
- Guide (quick win)
- Kit (deeper tools + scripts)
- Check On Me (ongoing support)

Bundle can combine top 3 matched guides, but each guide must still stand alone.
