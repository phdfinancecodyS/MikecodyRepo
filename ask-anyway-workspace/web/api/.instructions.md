# API Route Handler Instructions

## Route Architecture

These are Next.js App Router handlers. Each `route.ts` exports a `POST()` function. They must be copied into the target Next.js project at `app/api/`.

## 7 Endpoints — Do Not Collapse

- `POST /api/quiz/score` — Score + persist quiz session
- `POST /api/quiz/topic-match` — Match topic from post-quiz answers
- `POST /api/quiz/audience-match` — Assign audience bucket(s)
- `POST /api/quiz/recommendation` — Resolve guide + offer stack
- `POST /api/checkout/session` — Create Stripe checkout
- `POST /api/webhooks/stripe` — Purchase → fulfillment dispatch
- `POST /api/analytics/event` — Event tracking

## Contracts

Request/response shapes are defined in:
- `planning/API-ROUTE-SPECS.md` (full specs)
- `quiz/api-contracts.json` (machine-readable)
- `web/api/_lib/types.ts` (TypeScript interfaces)

## Shared Libraries

- `_lib/scorer.ts` — scoring logic (0-30 scale, Q5 overrides)
- `_lib/supabase.ts` — server client factory (service-role key, bypasses RLS)
- `_lib/types.ts` — all request/response TypeScript types

## Known TODOs

1. `quiz/topic-match/route.ts` — `matchDomain()` is a stub. Implement against `quiz/topic-matcher-flow.json`.
2. `quiz/recommendation/route.ts` — title resolver uses raw guide ID. Load from `quiz/base-guide-catalog.json`.
3. `webhooks/stripe/route.ts` — email/SMS fulfillment providers are stubs. Wire SendGrid + Twilio.
4. `analytics/event/route.ts` — non-CTA events log to stdout only. Persist to analytics store.

## Safety Rules

- `SUPABASE_SERVICE_KEY` is server-only. Never expose to browser.
- Stripe webhook signature verification is enforced. Never skip it.
- Critical risk users never receive paid offers before crisis resources.
