OVERVIEW
────────
This system auto-builds polished, fillable PDFs for each chapter of the
FrontlineFolks guidebook. You write the content in a simple JSON file,
run one command, and get two PDFs: a Standard version and a Clinical Depth
version.


FOLDER STRUCTURE
────────────────
  MikeNFS/
  ├── build_chapters.py       ← the builder script (never edit this)
  ├── INSTRUCTIONS.txt        ← this file
  ├── chapters/
  │   ├── _template/          ← master template — copy this for each chapter
  │   │   └── chapter.json
  │   ├── ch01/               ← Chapter 1 (already built by build_pdf.py)
  │   ├── ch02/               ← you create these
  │   │   └── chapter.json
  │   └── ch03/ ... ch47/
  └── output/                 ← every PDF also lands here for easy access


HOW TO CREATE A NEW CHAPTER
────────────────────────────
1. Open the chapters/ folder.
2. Copy the _template/ folder.
3. Rename the copy to the chapter folder name (e.g. ch02, ch03, etc.).
4. Open chapter.json inside the new folder.
5. Fill in the content (see FILLING IN CHAPTER.JSON below).
6. Run the builder (see RUNNING THE BUILDER below).


FILLING IN CHAPTER.JSON
────────────────────────
Open chapter.json in any text editor (Notepad, VS Code, etc.).
The file is JSON format — just edit the text between the quotes.

REQUIRED TOP-LEVEL FIELDS:
  "number"          — chapter number, e.g. 2
  "title"           — chapter title, e.g. "Sleep and the Frontline Brain"
  "cover_tagline"   — one short line under the subtitle on the cover

OPTIONAL TOP-LEVEL FIELDS:
  "series_subtitle" — defaults to "A Field Guide for Frontline Folks..."
  "build_standard"  — true/false, build the standard PDF (default: true)
  "build_clinical"  — true/false, build the clinical depth PDF (default: true)
  "disclaimer"      — custom disclaimer text (leave out to use default)


SECTIONS
────────
Each section in the "sections" array gets a numbered badge heading.

  {
    "number": 1,
    "heading": "Section Title",
    "content": [ ...content items... ]
  }


CONTENT TYPES (what goes inside "content": [...])
──────────────────────────────────────────────────

PARAGRAPH
  { "type": "para", "text": "Your paragraph text here." }

SUBHEADING (steel blue)
  { "type": "h2", "text": "Your Subheading" }

MINOR HEADING (dark bold)
  { "type": "h3", "text": "Your Minor Heading" }

BULLET LIST
  { "type": "bullets", "items": ["First bullet", "Second bullet"] }

  With a bold lead word:
  { "type": "bullets", "items": [
      {"lead": "Bold Word", "text": "regular text after it"}
  ]}

  Indented sub-bullets (level 1):
  { "type": "bullets", "level": 1, "items": ["Sub-bullet"] }

NUMBERED LIST
  { "type": "numbered", "items": ["First item", "Second item"] }

CALLOUT BOX
  Default (steel blue bar, light blue bg):
  { "type": "callout", "style": "default", "bold_first": true,
    "lines": ["Bold first line.", "", "Second line after a gap."] }

  Available styles:
    "default"    — steel blue bar, light blue background
    "highlight"  — same, use for key takeaways
    "info"       — light gray background, good for notes
    "clinical"   — green bar, very light bg. Use for "Why this works" notes.
                   Set "italic_first": true for these.

  Colour-coded blocks (Red/Yellow/Green system):
    "green"      — green left bar
    "yellow"     — amber left bar
    "red"        — red left bar
    "blue_box"   — steel blue, used for Blow Up / Shut Down contrast boxes

  TIP: Use an empty string "" in "lines" to add a small gap between lines.

DIVIDER (thin horizontal rule)
  { "type": "divider" }

PAGE BREAK
  { "type": "page_break" }

EXTRA SPACE
  { "type": "space", "pts": 10 }


CLINICAL-ONLY OR STANDARD-ONLY CONTENT
────────────────────────────────────────
Add "clinical_only": true to any content item to include it only in the
Clinical PDF. Add "standard_only": true to include it only in Standard.

Example:
  { "type": "h2", "text": "Why This Works", "clinical_only": true },
  { "type": "para", "text": "Clinical explanation...", "clinical_only": true }


WORKSHEETS
──────────
Each worksheet in the "worksheets" array becomes its own page.

QUESTION GROUPS (label + text fields):
  {
    "title": "Worksheet Title",
    "intro": "Instructions for the reader.",
    "question_groups": [
      {
        "label": "Question label shown above the fields",
        "hint": "Italicized hint text (optional)",
        "field_count": 3,         <- how many text boxes
        "field_height": 26,       <- height in points (26=one line, 48=two lines)
        "multiline": false        <- true allows multiple lines of typing
      }
    ]
  }

NUMBERED PROMPTS (repair script / step-by-step fill-in):
  {
    "title": "Worksheet Title",
    "intro": "Instructions.",
    "numbered_prompts": [
      {
        "step": "1.  Step label here:",
        "hint": "Prompt text in italics",
        "field_height": 48,
        "multiline": true
      }
    ]
  }

CLOSING NOTE on a worksheet:
  "closing_note": "Keep this somewhere private and update it as needed."


CLOSING SECTION
───────────────
The "closing" block at the bottom of chapter.json is optional.
If omitted, a default NFS closing is used.

  "closing": {
    "paragraphs": ["First paragraph.", "Second paragraph."],
    "highlight": "Key message that appears in a callout box."
  }


RUNNING THE BUILDER
────────────────────
Open a terminal / command prompt in the MikeNFS folder, then run:

  Build ALL chapters:
    python build_chapters.py

  Build ONE chapter:
    python build_chapters.py ch02

  List all chapters without building:
    python build_chapters.py --list

PDFs are saved in two places:
  1. Inside the chapter folder (e.g. chapters/ch02/NFS_Ch02_Standard.pdf)
  2. Inside the output/ folder (NFS_Ch02_Standard.pdf)


NAMING YOUR CHAPTER FOLDERS
────────────────────────────
Name them ch01, ch02 ... ch47 and they will sort in order automatically.
You can also use descriptive names like ch02_sleep if you prefer — the
chapter number shown in the PDF comes from the "number" field in chapter.json,
not the folder name.


TIPS
────
• JSON is picky about commas — every item except the last in a list
  needs a comma after it. If the builder crashes with a JSON error,
  check for a missing or extra comma.

• Use a JSON validator website (just search "JSON validator online") to
  check your file before building if you are unsure.

• The \u2014 code means an em dash (—). The \u2019 code means a smart
  apostrophe ('). You can also just type the characters directly.

• The \u2022 code means a bullet (•) — useful inside callout "lines".

• Leave the _readme and _note fields in place (or delete them — they are
  ignored by the builder). They are there to remind you what each section does.
