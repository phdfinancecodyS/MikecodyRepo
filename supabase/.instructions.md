# Supabase Migration Instructions

## Migrations (Apply in Filename Order)

1. `20260320153000_backend_architecture.sql` — creates all core tables
2. `20260320160000_rls_policies.sql` — Row Level Security policies

## Core Tables

- `leads` — person who starts/completes the quiz (email, phone, UTM, opt-ins)
- `quiz_sessions` — one quiz run (score, risk_level, override, answers JSON)
- `topic_match_sessions` — topic matcher answers and matched guide
- `audience_match_sessions` — audience bucket answers and selections
- `guide_recommendations` — resolved guide + offer stack per session
- `product_clicks` — CTA click tracking
- `purchases` — Stripe purchase records
- `fulfillment_events` — delivery tracking (email sent, SMS enrolled, etc.)

## Rules

- Do NOT recreate legacy `users` or `quiz_responses` tables
- Apply migration SQL exactly as written
- Schema reference: `planning/BACKEND-SYSTEM-CONTRACT.md`
- API handlers use service-role key (bypasses RLS) — see `web/api/_lib/supabase.ts`
